# Docs and Obsidian Memory

## Rules

- Update only docs affected by the change.
- Keep docs accurate, concise, and runnable.
- Prefer examples for setup and API usage.
- Record decisions separately from transient notes.
- Use links and headings that work in markdown/Obsidian.

## Useful Outputs

- README setup or usage update
- `docs/Architecture.md`
- `docs/Testing Strategy.md`
- ADR for a durable technical decision
- Session note with decisions, open questions, and next actions
